## Why

Текущий AI-автокомплит использует starcoder2-3b, который показывает посредственное качество предсказаний для shell-команд. Модель не специализирована на shell-скриптах и CLI-командах. Кроме того, контекст для AI-инференса ограничен только результатами compinit — файловая система и история команд полностью игнорируются, хотя именно они чаще всего определяют, какую команду пользователь хочет ввести (например, `cd` в конкретную директорию, повтор недавней команды, работа с файлами в текущей папке).

## What Changes

- **Смена модели на Qwen2.5-Coder** — специализированная модель для кода с лучшим качеством FIM (fill-in-the-middle), доступна в квантизованных GGUF-вариантах (1.5B, 3B, 7B). Конфигурация модели становится переключаемой, с автоопределением доступности.
- **Обогащение контекста AI данными файловой системы** — при вводе команды AI получает список файлов/директорий в текущей рабочей директории, что позволяет предсказывать аргументы-пути и команды, релевантные содержимому папки.
- **Обогащение контекста AI историей команд** — AI получает N последних команд из `.zsh_history`, что позволяет предсказывать повторяющиеся паттерны и цепочки команд.
- **Комбинированный промпт для AI** — новый формат промпта, объединяющий compinit-результаты, файлы и историю, с разделением по приоритету.
- **Опциональность источников контекста** — каждый источник (compinit, файлы, история) можно включать/отключать в конфигурации.
- **Автоопределение лучшей доступной модели** — поиск модели из приоритетного списка (qwen2.5-coder → starcoder2 → deepseek-coder); fallback на любую найденную.

## Capabilities

### New Capabilities

- `ai-model-selection`: Автоматический выбор лучшей доступной GGUF-модели из приоритетного списка. Поддержка переключения моделей без правки кода. Валидация наличия файла модели.
- `ai-context-enrichment`: Сбор и форматирование дополнительного контекста для AI-инференса: список файлов текущей директории, N последних команд из zsh-истории. Конфигурируемое включение/отключение источников.

### Modified Capabilities

- `ai-ghost-completion`: Изменяются требования к формату промпта — теперь он включает файловый и исторический контекст. Модель по умолчанию меняется со starcoder2-3b на автоопределение лучшей доступной.

## Impact

- **Конфигурация**: Новые поля в `AiConfig`: `modelPriority` (список путей к моделям), `fileContext` (bool + max files), `historyContext` (bool + max entries)
- **ai-completer.ts**: Изменение формата промпта, логики загрузки модели, добавление сбора контекста
- **index.ts**: Проброс новых опций при создании AiCompleter
- **zsh-completer.ts / новый модуль**: Чтение `.zsh_history` и `ls` текущей директории
- **Модель**: Пользователю потребуется скачать qwen2.5-coder GGUF (или использовать starcoder2 как fallback)
- **Тесты**: Обновление моков для нового формата промпта, тесты сбора контекста
